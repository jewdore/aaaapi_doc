## Г°ВџВ“ВҐ Step 2: Install GPT Proto Provider

<Steps>
  <Step title="Open Settings" icon="gear">
    Launch your chat application and navigate to the **Settings** or **Preferences** menu. Usually found in the top menu bar or sidebar.
  </Step>

  <Step title="Add New Provider" icon="plus">
    Look for options labeled:

  <Step title="Select GPT Proto" icon="check">

  </Step>
</Steps>

<Note>
  Г°ВџВ’ВЎ The exact menu names may vary between applications, but the concept remains the same: Add a custom API provider and configure it with GPT Proto's endpoint.
</Note>

## Г°ВџВ”В‘ Step 3: Enter API Credentials

<div style={{marginBottom: '24px'}}>
  Configure your connection by entering these two required parameters:
</div>

<ParamField path="Base URL" type="string" required>
  ```text
  https://gptproto.com/v1
  ```

  The base endpoint URL for all GPT Proto API requests. Make sure to copy this exactly!
</ParamField>

<ParamField path="API Key" type="string" required>
  ```text
  sk-your-api-key-here
  ```

  Your unique API key from the [GPT Proto Dashboard](https://gptproto.com/dashboard). Keep this secret and never share it publicly!
</ParamField>

### Г°ВџВ“В‹ Example Configuration

Here's what a typical configuration looks like in JSON format:

```json
{
  "provider": "GPT Proto",
  "baseURL": "https://gptproto.com/v1",
  "apiKey": "sk-your-api-key-here",
  "models": ["gpt-4", "gpt-3.5-turbo", "claude-3-opus"]
}
```

<Tip>
  Г°ВџВ”Вђ **Where to find your API Key?**

  Visit the [GPT Proto Dashboard](https://gptproto.com/dashboard) and navigate to the **API Keys** section. You can create new keys or manage existing ones there.
</Tip>

<Warning>
  ГўВљВ ГЇВёВЏ **Security Reminder**: Never commit API keys to version control or share them in public forums. Treat them like passwords!
</Warning>

## Г°ВџВ”В§ Troubleshooting

<AccordionGroup>
  <Accordion title="Г°ВџВ”Вґ Connection Error" icon="wifi">
    **Problem**: Cannot connect to the GPT Proto API

    **Solutions**:

    ГўВњВ… Verify your Base URL is exactly: `https://gptproto.com/v1` (no trailing slash)

    ГўВњВ… Check your internet connection is stable

    ГўВњВ… Ensure your firewall allows HTTPS connections to `gptproto.com`

    ГўВњВ… Try disabling VPN temporarily if you're using one

    ГўВњВ… Check if your corporate network blocks external APIs
  </Accordion>

  <Accordion title="Г°ВџВ”Вђ Authentication Failed (401)" icon="lock">
    **Problem**: Receiving "401 Unauthorized" error

    **Solutions**:

    ГўВњВ… Double-check your API key is copied correctly (no extra spaces)

    ГўВњВ… Verify your API key is active in the [Dashboard](https://gptproto.com/dashboard)

    ГўВњВ… Try regenerating a new API key if the old one doesn't work

    ГўВњВ… Make sure you're using the correct key format: `sk-...`

    ГўВњВ… Check if your API key has been revoked or expired
  </Accordion>

  <Accordion title="Г°ВџВ’Ві Insufficient Balance (402/403)" icon="credit-card">
    **Problem**: "Insufficient balance" or payment required error

    **Solutions**:

    ГўВњВ… Check your account balance in the [Dashboard](https://gptproto.com/dashboard)

    ГўВњВ… Add credits via the [Billing Center](https://gptproto.com/dashboard/billing-center)

    ГўВњВ… Review your usage limits and quotas

    ГўВњВ… Ensure your payment method is valid and up to date
  </Accordion>

  <Accordion title="Г°ВџВ¤В– Model Not Available (404)" icon="robot">
    **Problem**: Selected model is not responding or unavailable

    **Solutions**:

    ГўВњВ… Verify the model is available in your subscription plan

    ГўВњВ… Try switching to a different model (like GPT-3.5 Turbo)

    ГўВњВ… Check the [Status Page](https://status.gptproto.com) for service updates

    ГўВњВ… Ensure the model name is spelled correctly

    ГўВњВ… Some models may have regional restrictions
  </Accordion>

  <Accordion title="ГўВљВЎ Slow Responses" icon="clock">
    **Problem**: API responses are very slow

    **Solutions**:

    ГўВњВ… Try using a faster model like GPT-3.5 Turbo

    ГўВњВ… Check your internet connection speed

    ГўВњВ… Reduce the `max_tokens` parameter in your requests

    ГўВњВ… Consider your geographic distance from servers

    ГўВњВ… Check if there's high load on the service (status page)
  </Accordion>
</AccordionGroup>

## ГўВљВ ГЇВёВЏ Important Notes

<Warning>
  ### Г°ВџВ”В’ Security Best Practices

  ГўВЂВў **Never share your API key publicly** - Treat it like a password

  ГўВЂВў **Rotate keys regularly** - Generate new keys every few months

  ГўВЂВў **Use environment variables** - Never hardcode keys in your code

  ГўВЂВў **Monitor usage** - Check your dashboard regularly for unusual activity

  ГўВЂВў **Revoke compromised keys** - Immediately disable any exposed keys

  ГўВЂВў **Use separate keys** - Different keys for development and production
</Warning>

<Info>
  ### Г°ВџВЊВђ Network Requirements

  ГўВЂВў **Protocol**: HTTPS only (TLS 1.2 or higher)

  ГўВЂВў **Endpoint**: `https://gptproto.com/v1`

  ГўВЂВў **Authentication**: API Key required in headers

  ГўВЂВў **Firewall**: Ensure outbound HTTPS (port 443) is allowed

  ГўВЂВў **Proxy**: Configure if your network uses a proxy server

  ГўВЂВў **CORS**: Pre-configured for web applications
</Info>
