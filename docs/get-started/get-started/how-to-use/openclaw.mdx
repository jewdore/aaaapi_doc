## What is OpenClaw?

[OpenClaw](https://openclaw.ai) is a personal AI assistant that runs on your own device. It connects to various messaging platforms (Telegram, Discord, Slack, WhatsApp, etc.) and can be configured to use GPTProto's powerful AI models through our API.

<Tip>
  With GPTProto's unified API, OpenClaw can access **160+ AI models** from top providers including OpenAI, Google, Anthropic, DeepSeek, and more — all with a single API key.
</Tip>

## Install and Configure OpenClaw

<Steps>

<Step title="Get Your GPTProto API Key">

1. Visit [gptproto.com](https://gptproto.com) and sign up or log in.
2. Navigate to the **Dashboard** and create a new API Key (`sk-xxxxx`).
3. Keep this key secure — you'll need it in the next step.

<Warning>
  Keep your API key secure and never share it publicly. Treat it like a password.
</Warning>

</Step>

<Step title="Install OpenClaw">

<Note>
  For detailed installation guides, refer to the [official documentation](https://docs.openclaw.ai/zh-CN/install).
</Note>

**Prerequisites:** [Node.js 22 or newer](https://nodejs.org/en/download/)

The easiest way to install OpenClaw is using the official install script:

**macOS/Linux:**

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

**Windows (PowerShell):**

```powershell
iwr -useb https://openclaw.ai/install.ps1 | iex
```

</Step>

<Step title="Set Up OpenClaw">

After running the install command, the setup wizard will start automatically. If it doesn't, run:

```bash
openclaw onboard --install-daemon
```

If you've already initialized before, you can run `openclaw config` and select `model` to reconfigure.

Begin setup:

- `I understand this is powerful and inherently risky. Continue?` | Select ● `Yes`
- `Onboarding mode` | Select ● `Quick Start`
- `Model/auth provider` | Select ● `GPTProto`

</Step>

<Step title="Configure GPTProto Provider">

After selecting `GPTProto` as the provider, the system will prompt you to enter your API Key.

Paste your GPTProto API Key and press Enter, then select the model you want to use.

**Recommended models:**

| Model | Description | Best For |
|-------|-------------|----------|
| `gptproto/claude-opus-4-6` | Anthropic Claude Opus 4 | Complex reasoning, coding |
| `gptproto/gpt-5.2` | OpenAI GPT-5.2 | General tasks, creative writing |
| `gptproto/gemini-3-pro` | Google Gemini 3 Pro | Multimodal, image understanding |
| `gptproto/deepseek-v3.2` | DeepSeek V3.2 | Code generation, math |
| `gptproto/claude-sonnet-4-6` | Anthropic Claude Sonnet 4 | Fast, balanced performance |

<Tip>
  You can switch models at any time using the `/model` command in your chat, or by updating your configuration.
</Tip>

</Step>

<Step title="Complete Setup">

Continue with the remaining OpenClaw configuration:

- `Select channel` | Choose and configure your preferred messaging channel.
- `Configure skills` | Select and install the skills you need.
- Complete the setup.

</Step>

<Step title="Chat with Your Bot">

After setup, the CLI will ask `How do you want to hatch your bot?`

- Select ● `Hatch in TUI (recommended)`

Now you can start chatting with your AI bot in the Terminal UI!

OpenClaw supports many channels for interaction, including Web UI, Discord, Slack, Telegram, and more. Refer to the official docs for channel setup: [Channels Setup](https://docs.openclaw.ai/channels/setup)

- For Web UI, open the `Web UI (with token)` link shown in your terminal.

</Step>

<Step title="Verify Installation">

Verify everything is working correctly:

```bash
openclaw doctor     # Check configuration issues
openclaw status     # View gateway status
openclaw dashboard  # Open Dashboard in browser
```

</Step>

</Steps>

<Note>
  For detailed configuration guides, refer to the [official documentation](https://docs.openclaw.ai/zh-CN/start/getting-started).
</Note>

<Warning>
  OpenClaw may involve security risks if improperly configured or deployed without proper access controls. Refer to the [official security documentation](https://docs.openclaw.ai/gateway/security).
</Warning>

## Advanced Configuration

### Manual Configuration

If you prefer to configure manually, edit `~/.openclaw/openclaw.json`:

```json
{
  "models": {
    "providers": {
      "gptproto": {
        "apiKey": "sk-your-api-key-here",
        "baseUrl": "https://gptproto.com/v1"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "gptproto/claude-opus-4-6"
      }
    }
  }
}
```

### Model Failover

Configure model failover to ensure reliability:

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "gptproto/gpt-5.2",
        "fallbacks": ["gptproto/claude-opus-4-6", "gptproto/gemini-3-pro", "gptproto/deepseek-v3.2"]
      }
    }
  }
}
```

<Tip>
  With GPTProto's unified API, failover works across different providers seamlessly. If GPT-5.2 is unavailable, it automatically switches to Claude, Gemini, or DeepSeek.
</Tip>

### Using Skills

> Skills are folders containing a `SKILL.md` file. If you want to add new capabilities to your OpenClaw agent, [ClawHub](https://clawhub.ai/) is the best way to find and install skills.

#### Install ClawHub

```bash
npm i -g clawhub
```

#### Manage Skills

Search for skills:

```bash
clawhub search "postgres backups"
```

Install a new skill:

```bash
clawhub install my-skill-pack
```

Update installed skills:

```bash
clawhub update --all
```

### Plugin Management

> Plugins are small code modules that extend OpenClaw with additional capabilities (commands, tools, and Gateway RPC).

View loaded plugins:

```bash
openclaw plugins list
```

Install official plugins (example: voice-call):

```bash
openclaw plugins install @openclaw/voice-call
```

Restart the gateway:

```bash
openclaw gateway restart
```

## Troubleshooting

### Common Issues

- **API Key Authentication**
  - Ensure your GPTProto API key is valid and has sufficient balance
  - Check that the API key is correctly set in your configuration
  - Verify at [gptproto.com/dashboard](https://gptproto.com/dashboard)

- **Model Availability**
  - Check available models at [gptproto.com/v1/models](https://gptproto.com/v1/models)
  - Verify the model name format (e.g., `gptproto/gpt-5.2`)

- **Connection Issues**
  - Ensure the OpenClaw gateway is running: `openclaw status`
  - Check network connectivity to `gptproto.com`
  - Run `openclaw doctor` for diagnostics
