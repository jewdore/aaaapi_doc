## Overview

This endpoint generates images based on input images using Gemini Flash Image model via the official Gemini API format. It supports image-to-image transformation with text prompts.

## Authentication

This endpoint requires authentication using an API key.

<ParamField header="Authorization" type="string" required default="sk-***********">
  Your API key
</ParamField>

## Path Parameters

<ParamField path="model" type="string" required default="gemini-25-flash-image-preview">
  The model to use for image generation. Available models: `gemini-25-flash-image-preview`, etc.
</ParamField>

## Request Body Parameters

<ParamField body="contents" type="array" required>
  An array of content objects representing the generation prompt and input image.

  <Expandable title="Content Object Structure">
    <ParamField body="role" type="string">
      The role of the message sender. Typically `"user"`. Optional for single-turn requests.
    </ParamField>

    <ParamField body="parts" type="array" required>
      An array of parts that make up the message content, including text prompt and input image.

      <Expandable title="Parts Object Structure">
        <ParamField body="text" type="string">
          Text prompt for image transformation.
        </ParamField>

        <ParamField body="inline_data" type="object">
          Input image data in base64 format.

          <Expandable title="Inline Data Structure">
            <ParamField body="mime_type" type="string" required>
              The MIME type of the image (e.g., `"image/jpeg"`, `"image/png"`).
            </ParamField>

            <ParamField body="data" type="string" required>
              Base64-encoded image data.
            </ParamField>
          </Expandable>
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="generationConfig" type="object">
  Configuration for the image generation behavior.

  <Expandable title="Generation Config Options">
    <ParamField body="responseModalities" type="array">
      Response modalities. For image generation, use `["TEXT", "IMAGE"]`.
    </ParamField>

    <ParamField body="imageConfig" type="object">
      Image-specific configuration.

      <Expandable title="Image Config Options">
        <ParamField body="aspectRatio" type="string">
          Aspect ratio for the generated image. Examples: `"16:9"`, `"1:1"`, `"9:16"`.
        </ParamField>
      </Expandable>
    </ParamField>
  </Expandable>
</ParamField>

```python Python
import requests
import json
import base64

url = "https://gptproto.com/v1beta/models/gemini-25-flash-image-preview:generateContent"
headers = {
    "Authorization": "sk-***********",
    "Content-Type": "application/json"
}

# Read and encode image
with open("input_image.jpg", "rb") as image_file:
    image_data = base64.b64encode(image_file.read()).decode('utf-8')

data = {
    "contents": [
        {
            "role": "user",
            "parts": [
                {
                    "text": "Create a picture of my cat eating a nano-banana in a fancy restaurant under the Gemini constellation"
                },
                {
                    "inline_data": {
                        "mime_type": "image/jpeg",
                        "data": image_data
                    }
                }
            ]
        }
    ],
    "generationConfig": {
        "responseModalities": [
            "TEXT",
            "IMAGE"
        ]
    }
}

response = requests.post(url, headers=headers, json=data)
result = response.json()
print(json.dumps(result, indent=2))
```

    if err != nil {

    defer resp.Body.Close()

    body, _ := ioutil.ReadAll(resp.Body)
